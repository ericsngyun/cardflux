## Critical Bugs Found & Fixed

### 🔴 BUG #1: Broken Python Imports in identification_service.py

**Severity**: CRITICAL - App cannot start
**File**: `apps/desktop/src/python/identification_service.py`
**Commit**: f9ec6b4

**Problem**:
```python
# BROKEN - No package structure
from production_card_identifier import ProductionCardIdentifier
from card_detector import StabilizedCardDetector
from identifier_version_manager import IdentifierVersionManager
from capture.capture_manager import CaptureManager
```

**Issues**:
1. Imports don't use package structure (`core.`, `tools.`)
2. Wrong class name (`StabilizedCardDetector` doesn't exist)
3. Wrong filename (`card_detector.py` vs `polished_card_detector.py`)
4. Non-existent module path (`capture/capture_manager.py`)

**Fix**:
```python
# FIXED - Proper package imports
from core.production_card_identifier import ProductionCardIdentifier
from core.polished_card_detector import PolishedCardDetector, CardDetectionStatus
from tools.identifier_version_manager import IdentifierVersionManager

# Graceful fallback for missing module
try:
    from tools.capture_manager import CaptureManager
except ImportError:
    CaptureManager = None
    print("[PY] Warning: CaptureManager not available (module not found)", file=sys.stderr)
```

**Impact**: Service can now import all required modules
**Test**: ✅ Service starts without ModuleNotFoundError

### 🔴 BUG #4: JSON-RPC Pollution from print() Statements

**Severity**: CRITICAL - JSON-RPC communication broken
**File**: `scripts/identification/tools/identifier_version_manager.py`
**Commit**: e69c80a

**Problem**:
```python
# BROKEN - Prints to stdout, pollutes JSON-RPC
print(f"[VersionManager] Initialized (default: {default_version}...)")
print(f"[VersionManager] Loading v1 (baseline) identifier...")
print(f"[VersionManager] V2 low confidence, trying V1 fallback...")
# ... 8 more print() statements
```

**Issue**:
- Version manager printed to stdout (lines 83, 100, 111, 113, 176, 194, 201, 226, 248, 328, 339)
- JSON-RPC uses stdout for communication
- Print statements broke JSON parsing on Electron side
- Service appeared to hang or respond with garbage

**Fix**:
```python
# FIXED - All logging to stderr
def _log(message: str):
    """Log to stderr to avoid interfering with JSON-RPC on stdout."""
    print(f"[VersionManager] {message}", file=sys.stderr, flush=True)

# Replace all print() with _log()
_log("Initialized (default: {default_version}...)")
_log("Loading v1 (baseline) identifier...")
```

**Impact**: JSON-RPC communication now works correctly
**Test**: ✅ Service responds to `initialize` and `status` JSON-RPC requests

### ⚠️ BUG #5: Missing CaptureManager Module (Non-Fatal)

**Severity**: MEDIUM - Feature unavailable, but handled gracefully
**File**: `apps/desktop/src/python/identification_service.py`
**Commit**: f9ec6b4

**Problem**:
- Service expected `tools/capture_manager.py` to exist

**Fix**:

```python
# Handle None case during initialization
if CaptureManager is not None:
    self.capture_manager = CaptureManager(game=game)
else:
    self.capture_manager = None
    self.auto_capture = False  # Disable auto-capture
```

**Impact**: Service starts successfully without capture manager
**Test**: ✅ Service initializes with warning, auto-capture disabled

## Lessons Learned

### 1. **Import Structure Critical**
- Python package structure must be consistent
- `__init__.py` files are NOT optional for package imports
- Always test imports from different working directories

### 2. **JSON-RPC Requires Discipline**
- stdout is SACRED for JSON-RPC communication
- All logging must go to stderr
- One stray print() breaks everything

### 3. **Graceful Degradation**
- Missing modules should be handled gracefully
- Use try/except for optional imports
- Log warnings clearly to stderr

### 4. **Path Resolution Matters**
- Use `pathlib.Path` for cross-platform compatibility
- Test path resolution from multiple working directories
- `Path(__file__).parent.parent...` works but is fragile

### 5. **Testing is Non-Negotiable**
- Must test imports in isolation
- Must test JSON-RPC communication end-to-end
- Must test on clean environment (VM)
